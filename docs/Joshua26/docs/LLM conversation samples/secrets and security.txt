An attacker would have to compromise both your code repository and your deployment pipeline at the exact same time to spoof the agent.3. It completely eliminates "Secret Zero" because the machine doesn't use a password at all. It uses cryptographic trust.How it works: The secrets manager is configured to trust the underlying infrastructure where the agent lives.If your agent is running in a Kubernetes pod, Kubernetes automatically injects a cryptographically signed, invisible "badge" (a JWT token) into the pod.When the agent wakes up, it hands that badge to the secrets manager.The secrets manager pings the Kubernetes server and asks, "Did you sign this badge for Agent A?" 4.
