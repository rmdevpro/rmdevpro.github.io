  1. The HuggingFace Cache (Existing)                                                                                                                                                         
  Name: Alexandria                                                                                                                                                                            
                                                                                                                                                                                              
  What it Caches: Large Models (GBs/TBs).                                                                                                                                                     
                                                                                                                                                                                              
  Mechanism: Transparent Proxy (Olah).                                                                                                                                                        
                                                                                                                                                                                              
  Client Config: HF_ENDPOINT=http://alexandria:8090                                                                                                                                           
                                                                                                                                                                                              
  Storage Location: /storage/models/alexandria on the Irina server.                                                                                                                           
                                                                                                                                                                                              
  2. The NPM Cache (New Requirement)                                                                                                                                                          
  Name: (Proposed) "NPM-Mirror" (Verdaccio)                                                                                                                                                   
                                                                                                                                                                                              
  What it Caches: Node.js packages (express, react, etc.).                                                                                                                                    
                                                                                                                                                                                              
  Mechanism: Transparent Proxy (Verdaccio). It looks exactly like the real NPM registry to your agents.                                                                                       
                                                                                                                                                                                              
  Client Config: npm config set registry http://npm-mirror:4873                                                                                                                               
                                                                                                                                                                                              
  Why it matches Alexandria:                                                                                                                                                                  
                                                                                                                                                                                              
  First request: Downloads from npmjs.org, saves to disk, serves to agent.                                                                                                                    
                                                                                                                                                                                              
  Subsequent requests: Serves instantly from disk.                                                                                                                                            
                                                                                                                                                                                              
  Offline capable.                                                                                                                                                                            
                                                                                                                                                                                              
  3. The PyPI Cache (New Requirement)                                                                                                                                                         
  Name: (Proposed) "PyPI-Mirror" (Devpi or simple Nginx cache)                                                                                                                                
                                                                                                                                                                                              
  What it Caches: Python wheels (pandas, torch, numpy).                                                                                                                                       
                                                                                                                                                                                              
  Mechanism: Transparent Proxy.                                                                                                                                                               
                                                                                                                                                                                              
  Client Config: pip config set global.index-url http://pypi-mirror:3141/root/pypi/+simple/                                                                                                   
                                                                                                                                                                                              
  Why it matches Alexandria:                                                                                                                                                                  
                                                                                                                                                                                              
  Prevents the agents from running pip download manually (which is slow and complex).                                                                                                         
                                                                                                                                                                                              
  Centralizes all python wheels on your storage server.

  You are building a "Supply Chain Wall" around your lab. Nothing enters the lab twice.

  Why this is better than your current ADR                                                                                                                                                    
  Your current ADR-037 requires the Agent to manually manage the cache ("download this to a folder, then install from that folder").                                                          
                                                                                                                                                                                              
  With this Network Proxy approach:                                                                                                                                                           
                                                                                                                                                                                              
  Zero Agent Cognitive Load: The agent just runs pip install pandas. Your current ADR isolates caches per-project, leading to duplication.            
                                                                                                                                                                                              
  Strict Compliance: You configure the base Docker images to hardcode these environment variables. The agents literally cannot access the public internet, forcing them to use the cache.

❯ next let's talk about CI/CD because I think that will be the natural follow-on to the caching model. Here is the conversation that I just had with Gemini: Here is your consolidated CI/CD Strategy 2.0.                                                                                                                                                                         
                                                                                                                                                                                              
  This strategy replaces the "Folder Copying" model (ADR-037) with the "Library Proxy" model. It leverages GitHub for code storage but keeps all heavy lifting (builds and packages) on       
  your local network to support your AI Lab environment.                                                                                                                                      
                                                                                                                                                                                              
  Strategy Name: The "Hybrid-Local" Pipeline                                                                                                                                                  
  The Core Concept:                                                                                                                                                                           
                                                                                                                                                                                              
  Brains (Agents): Operate locally on file shares.                                                                                                                                            
                                                                                                                                                                                              
  Record (Git): Lives on GitHub.com (Cloud).                                                                                                                                                  
                                                                                                                                                                                              
  Muscle (Builds): Runs on Self-Hosted Runners in your Lab.                                                                                                                                   
                                                                                                                                                                                              
  Supply (Cache): Served by local proxies (Alexandria, Delphi, Pergamon).                                                                                                                     
                                                                                                                                                                                              
  1. The Architecture                                                                                                                                                                         
  A. The Cloud Layer (Control Plane)                                                                                                                                                          
  GitHub Repository: Stores source code, Issues, and Pull Requests.                                                                                                                           
                                                                                                                                                                                              
  GitHub Actions: Orchestrates the workflow (decides when to run).                                                                                                                            
                                                                                                                                                                                              
  Role: The "Source of Truth" and backup.                                                                                                                                                     
                                                                                                                                                                                              
  B. The Lab Layer (Data Plane)                                                                                                                                                               
  Self-Hosted Runners: Small containers on your local server that listen for jobs from GitHub.                                                                                                
                                                                                                                                                                                              
  Why: They sit inside your firewall, so they can see your local proxies.                                                                                                                     
                                                                                                                                                                                              
  The "Library" Stack (Caching Proxies):                                                                                                                                                      
                                                                                                                                                                                              
  Alexandria (8090): HuggingFace Models (Olah).                                                                                                                                               
                                                                                                                                                                                              
  Delphi (3141): Python PyPI (Devpi).                                                                                                                                                         
                                                                                                                                                                                              
  Pergamon (4873): Node NPM (Verdaccio).                                                                                                                                                      
                                                                                                                                                                                              
  The Gatekeeper (MCP Server): The API that Agents use to interact with Git safely.                                                                                                           
                                                                                                                                                                                              
  2. The Agent Workflow (The "Happy Path")                                                                                                                                                    
  Here is the lifecycle of a task under this new strategy:                                                                                                                                    
                                                                                                                                                                                              
  Phase 1: The Local Work (Fast & Cheap)                                                                                                                                                      
                                                                                                                                                                                              
  Task: Agent receives: "Update the data processing pipeline."                                                                                                                                
                                                                                                                                                                                              
  Edit: Agent reads files directly from the local file share (0 API cost).                                                                                                                    
                                                                                                                                                                                              
  Install: Agent needs a new library. It calls the MCP tool install_package.                                                                                                                  
                                                                                                                                                                                              
  Under the hood: The MCP server runs pip install pandas.                                                                                                                                     
                                                                                                                                                                                              
  The Magic: The server is configured to hit Delphi (http://delphi:3141). If Delphi has it, it downloads instantly. If not, Delphi fetches it once from the internet.                         
                                                                                                                                                                                              
  Test: Agent runs tests locally using the cached packages.                                                                                                                                   
                                                                                                                                                                                              
  Phase 2: The Safe Commit (Sanitized) 5. Commit: Agent calls MCP tool git_commit. * Check: MCP ensures the agent isn't committing to main or deleting protected files. 6. Push: Agent        
  calls MCP tool git_push. * Action: Code goes to GitHub.com.                                                                                                                                 
                                                                                                                                                                                              
  Phase 3: The Automated Build (The Robot) 7. Trigger: Agent calls MCP tool create_pull_request. 8. Action: GitHub sees the PR and signals your Self-Hosted Runner. 9. Build: The Runner      
  spins up a container. * It pulls code from GitHub. * It sets env vars: PIP_INDEX_URL=http://delphi:3141 and NPM_CONFIG_REGISTRY=http://pergamon:4873. * Result: The build is lightning      
  fast because it pulls packages from the local proxies, never touching the public internet.                                                                                                  
                                                                                                                                                                                              
  3. Key Configuration Changes                                                                                                                                                                
  To make this work, you need to "poison" the environment variables so tools only look at your local libraries.                                                                               
                                                                                                                                                                                              
  For Python (Delphi):                                                                                                                                                                        
                                                                                                                                                                                              
  File: ~/.pip/pip.conf (on Agents and Runners)                                                                                                                                               
                                                                                                                                                                                              
  Setting: index-url = http://delphi:3141/root/pypi/+simple/                                                                                                                                  
                                                                                                                                                                                              
  For Node (Pergamon):                                                                                                                                                                        
                                                                                                                                                                                              
  File: ~/.npmrc (on Agents and Runners)                                                                                                                                                      
                                                                                                                                                                                              
  Setting: registry = http://pergamon:4873/                                                                                                                                                   
                                                                                                                                                                                              
  For Models (Alexandria):                                                                                                                                                                    
                                                                                                                                                                                              
  Env Var: HF_ENDPOINT = http://alexandria:8090                                                                                                                                               
                                                                                                                                                                                              
  4. What this Replaces (The "Kill List")                                                                                                                                                     
  Kill: The packages/ folder inside every app directory (ADR-037). You no longer need to commit tarballs to Git.                                                                              
                                                                                                                                                                                              
  Kill: The pip download step in your Agent instructions. They can just "install" normally, and the network layer handles the caching.                                                        
                                                                                                                                                                                              
  Kill: GitHub Cloud Runners. They are too slow and can't see your cached data.

  What is a Docker Registry?                                                                                                                                                                  
                                                                                                                                                                                              
  A Docker registry stores built container images (the .tar files that docker build produces). It's the "package repository" for containers.                                                  
  ┌─────────────────┬────────────────────────┬──────────────────────────────┐                                                                                                                 
  │ Repository Type │         Stores         │           Examples           │

  │ Pergamum (NPM)  │ Node packages (.tgz)   │ express, react               │                                                                                                                 
  ├─────────────────┼────────────────────────┼──────────────────────────────┤                                                                                                                 
  │ Docker Registry │ Container images       │ joshua:latest, delphi:1.0    │                                                                                                                 
  └─────────────────┴────────────────────────┴──────────────────────────────┘

  BOMs (Bill of Materials)                                                                                                                                                                    
                                                                                                                                                                                              
  BOMs are not stored in the registry by default. They're metadata about the images:                                                                                                          
  ┌─────────────────────┬──────────────────────────────────────────────────────────────────┬───────────────────────────────────────┐                                                          
  │      BOM Type       │                          What It Lists                           │                Purpose                │                                                          
  ├─────────────────────┼──────────────────────────────────────────────────────────────────┼───────────────────────────────────────┤                                                          
  │ SBOM (Software BOM) │ All packages inside the image (pandas 2.1.0, numpy 1.26.0, etc.) │ Security auditing, license compliance │                                                          
  ├─────────────────────┼──────────────────────────────────────────────────────────────────┼───────────────────────────────────────┤                                                          
  │ Build BOM           │ Source commit, build time, base image used                       │ Reproducibility                       │                                                          
  ├─────────────────────┼──────────────────────────────────────────────────────────────────┼───────────────────────────────────────┤                                                          
  │ Dependency BOM      │ What this image depends on (base images, etc.)                   │ Supply chain tracking                 │                                                          
  └─────────────────────┴──────────────────────────────────────────────────────────────────┴───────────────────────────────────────┘                                                          
  Where BOMs Can Live                                                                                                                                                                         
                                                                                                                                                                                              
  1. Attached to images (OCI artifacts) - Harbor and modern registries support this                                                                                                           
  2. Separate files in Git - e.g., sbom.json committed alongside Dockerfile                                                                                                                   
  3. Dedicated SBOM server - enterprise pattern                                                                                                                                               
  4. Generated on-demand - scan images when needed
