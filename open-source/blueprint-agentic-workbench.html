§ 01 · Why

One browser, every AI.

Every AI coding tool today runs in its own terminal, with its own state, with its own context window, with its own CLI conventions. If you work with more than one, you end up juggling tmux windows, lost sessions, and copy-paste between chats.

Blueprint Agentic Workbench is what happens when you take that pain seriously. One browser, multiple AI agents, proper project boundaries, persistent sessions, real file browsing, and integrated task tracking — all self-hosted in Docker, single-user by design.

§ 02 · Capabilities

What it does.

  • Multi-AI session management — Create, pause, resume, and switch between Claude, Gemini, and Codex sessions. State persists across restarts.
  • Browser-based terminal — Full terminal access over WebSocket, powered by xterm.js. No native install, no desktop app.
  • Project organization — Group sessions by project. Each gets its own workspace, configuration, and MCP tool registry.
  • Integrated task tracking — Create and organize tasks inline with your work. Surface what's in-flight at a glance.
  • Vector search — Built-in Qdrant vector database for semantic search across your codebase.
  • Workspace file browser — Browse, create, edit, and upload files directly from the web UI.
  • MCP tool integration — Register Model Context Protocol servers per project.
  • Environment safety markers — Deploy as Dev (green) or Pro (red) to prevent prod/dev confusion at a glance.
§ 03 · Who it's for

For builders, not evaluators.

If you juggle multiple AI coding tools across multiple projects, and you've accepted that tmux pain as normal — this is for you.

  • Solo developers running several AI agents on different problems in parallel
  • Teams evaluating different models side-by-side under identical conditions
  • Anyone who wants persistent, reproducible agent sessions without chasing terminal windows
§ 04 · Stack

Under the hood.

Node.js 22 · Express · SQLite · tmux · node-pty · xterm.js · Qdrant · Docker. No build step in production, no external cloud dependencies, no telemetry.

§ 05 · Install

Run it yourself.

One command. You need Docker and docker compose.

# Clone and run git clone https://github.com/rmdevpro/blueprint.git cd blueprint docker compose up -d

Open http://localhost:8080. That's it.
