§ Agents · Open source

Pluggable Agents.

Drop-in agents for the services your business already runs on.

Every eMAD (ephemeral Modular Agent Daemon) is a Python package that plugs into an eMAD Host. Install it, the Host picks it up, and agents get new capabilities — no downtime, no monolithic redeploy. Two agents are currently published.

google-workspace-emadv0.2

Google Workspace

Gmail · Drive · Docs · Sheets · Slides · Calendar. 30+ tools.

microsoft-365-emadv0.2

Microsoft 365

Outlook · Calendar · OneDrive. 15+ tools via Microsoft Graph.

§ 01 · Google Workspace

Full Google Workspace, agent-accessible.

Gmail, Drive, Docs, Sheets, Slides, and Calendar — all callable by agents via a single registered eMAD. Service-account authentication with domain-wide delegation means agents act on behalf of real users, safely.

Capabilities

Gmailread · send · search
Drivelist · search · create · delete · share · move · copy · rename
Sheetsread · write · append · clear · create
Docsread · write · create
Slidesget · create
Calendarlist · create · update · get events

Stack

Python 3.12 · LangGraph · LangChain · google-api-python-client · asyncpg · Prometheus.

§ 03 · Why "pluggable"

Install and the host picks it up.

Because agents register with their host via Python entry points, you can:

  • Install a new agent with pip install — the host discovers it automatically
  • Upgrade an agent without restarting the host
  • Develop agents independently from the runtime they'll be hosted in
  • Compose agents — a workflow can call Google Workspace, then Microsoft 365, then a custom internal agent, all in one conversation
§ 04 · Install

Run them yourself.

# Requires eMAD Host already running pip install google-workspace-emad pip install microsoft-365-emad # Restart not required — host discovers on install

Documentation for each agent lives in its repository.
