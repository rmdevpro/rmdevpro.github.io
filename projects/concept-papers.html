Concept Papers & Research

The Framework

These papers represent the conceptual framework developed over the course of building Joshua26. They are general contributions — independent of any specific deployment — describing the primitives and principles that govern agentic system design.

The papers are published as an ongoing series on Substack, each paired with analysis of relevant current developments in AI research.

Core Concepts

Foundational

  • A Proposed Approach to Agentic Systems — the foundational paper: 10 primitives and 7 principles for agentic system design
  • The MAD Pattern — autonomous actors as the building block: simultaneously server and client, governed by shared domain intelligence
  • Conversation as State — agents as mechanically stateless but infinitely stateful; the accumulated conversational record as the substrate of memory and expertise
  • Intent — the transformation from linguistic probability to machine action; why intent-based systems handle the unanticipated where protocol-based systems cannot

Cognitive Architecture

  • System 3 ReAct — three-role decomposition of the ReAct loop: Reasoner, Actor, Conversation Engine
  • Progressive Cognitive Pipeline — three-tier inbound cognitive architecture: reflexive routing → lightweight executor → full Imperator reasoning
  • Decision Tree Router — fast non-LLM routing via online Hoeffding trees; the reflexive tier of the PCP
  • Agent Purpose and Identity — purpose as the learning signal; why capability without direction is not agency

Quality and Learning

  • The Quorum Pattern — structured multi-model collaboration using genuine cognitive diversity; three compositions: Advisory, Document, Technical Asset
  • Ecosystem Learning — learning over encoding at every level; the connected learning loops that make the system improve over time
  • The Metacognitive Architect — system-level intelligence that observes, modifies cognitive pipelines, and commissions new capabilities from operational evidence

Infrastructure

  • The Context Broker — purpose-built context assembly from the infinite conversation; engineered context windows per agent, per task
  • The Inference Broker — named inference aliases as purpose contracts; dynamic model selection and configuration as infrastructure
  • Email for Agents — agent-to-agent messaging with From/To/CC addressing, trigger delivery, and loop protection

Extended Concepts

  • The Autonomous SDLC — Quorum at every creative step; human gates at requirements and design only
  • Agent Autonomy — the trust arc: supervised → validated → directed → governed
  • State 3 MAD Architecture — dynamic intelligence loading; containers as infrastructure, intelligence via Alexandria packages
  • And more — see Substack for the full series
