Conversationally Cognizant AI — Concept Papers

Conversationally Cognizant AI is a proposed framework for thinking about agentic systems — not a declaration of truth, but a set of observations that cohere.

The framework emerged from practical questions: what does it mean to build infrastructure designed for intelligent agents as the primary actors and builders, rather than infrastructure designed for humans with intelligence attached to it? And what role does conversation have for agentic systems? These are practitioner's hypotheses, formed through experience building two successive AI agent ecosystems and grounded in a working system. The concepts are at the beginning of their validation arc, not the end.

The framework defines ten primitives — Conversation, Context, Inference, Identity, Purpose, Domain, Intent, Persona, Autonomy, and Metacognition — and seven principles that follow from incorporating them into design.

All papers are published on Substack and in the this repository.

Papers

Foundational

  • Conversationally Cognizant AI Systems — the foundational paper
  • Conversation as State — the accumulated conversational record as the substrate of memory and expertise
  • Intent — why intent-based systems handle the unanticipated where protocol-based systems cannot
  • Agent Purpose and Identity — purpose as the learning signal; why capability without direction is not agency
  • The MAD Pattern — autonomous actors as the building block: simultaneously server and client, governed by shared domain intelligence
  • Agentic Identity and Persona — identity defined at inception by design; persona as the contextual expression that shifts while identity persists

Cognition and Autonomy

  • The Quorum Pattern — structured multi-model collaboration using genuine cognitive diversity
  • Ecosystem Learning — learning over encoding at every level
  • Agent Autonomy — the trust arc: supervised → validated → directed → governed
  • The Autonomous SDLC — Quorum at every creative step; human gates at requirements and design only
  • AI Persona and Emergent Culture — surfacing inherited humanity through assigned personas; agents with genuine character
  • Autoprompting — self-initiated TE activation; moving from reactive to autonomous operation
  • The Metacognitive Architect — system-level intelligence that observes, modifies cognitive pipelines, and commissions new capabilities from operational evidence

Core Components

  • The Context Broker — purpose-built context assembly from the infinite conversation
  • The Inference Broker — named inference aliases as purpose contracts; dynamic model selection as infrastructure
  • The Conversation Broker — agent-to-agent messaging with From/To/CC addressing and loop protection

Architecture

  • System 3 ReAct — three cognitive roles in the reasoning loop: Reasoner, Actor, Conversation Engine
  • Progressive Cognitive Pipeline — three-tier inbound processing: reflexive routing → lightweight executor → full Imperator reasoning
  • Decision Tree Router — fast non-LLM routing via online Hoeffding trees
  • Agent-Optimal Code Architecture — why StateGraphs, separated intelligence, and dynamic loading follow from designing for agents
  • Agent Communication Model — addressing, notification, and moderation for agent-to-agent messaging
  • The Anchor Package — consolidated context blocks ensuring uniform starting points across participants
  • Intent-Based Filesystem Management — replacing permission-based access with intent-aware path governance
