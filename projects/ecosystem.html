CCAI Ecosystem Components

These are standalone tools extracted from the Joshua26 ecosystem. Each one implements concepts from the Conversationally Cognizant AI framework as deployable software that anyone can run. Every component works independently — docker compose up on any machine with Docker installed. No ecosystem required.

Each tool is configurable: point it at OpenAI, Anthropic, Gemini, Ollama, or any OpenAI-compatible endpoint. Same code runs standalone or inside a larger ecosystem — only the configuration changes.

What is a State 4 MAD?

Every component published here is a State 4 MAD — the final step in an architectural evolution where every ecosystem dependency becomes a configuration choice. The same code runs on a laptop, a cloud VM, or inside a full ecosystem. Only config.yml changes between environments. See Agent-Optimal Code Architecture for the full evolution arc.
