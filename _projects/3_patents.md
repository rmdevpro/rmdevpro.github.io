## The Framework

These papers represent the conceptual framework developed over the course of building Joshua26. They are general contributions — independent of any specific deployment — describing the primitives and principles that govern agentic system design.

The papers are published as an ongoing series on the [Blog](/blog/), each paired with analysis of relevant current developments in AI research.

## Core Concepts

**Foundational**
- [A Proposed Approach to Agentic Systems](/blog/) — the foundational paper: 10 primitives and 7 principles for agentic system design
- [The MAD Pattern](/blog/) — autonomous actors as the building block: simultaneously server and client, governed by shared domain intelligence
- [Conversation as State](/blog/) — agents as mechanically stateless but infinitely stateful; the accumulated conversational record as the substrate of memory and expertise
- [Intent](/blog/) — the transformation from linguistic probability to machine action; why intent-based systems handle the unanticipated where protocol-based systems cannot

**Cognitive Architecture**
- [System 3 ReAct](/blog/) — three-role decomposition of the ReAct loop: Reasoner, Actor, Conversation Engine
- [Progressive Cognitive Pipeline](/blog/) — three-tier inbound cognitive architecture: reflexive routing → lightweight executor → full Imperator reasoning
- [Decision Tree Router](/blog/) — fast non-LLM routing via online Hoeffding trees; the reflexive tier of the PCP
- [Agent Purpose and Identity](/blog/) — purpose as the learning signal; why capability without direction is not agency

**Quality and Learning**
- [The Quorum Pattern](/blog/) — structured multi-model collaboration using genuine cognitive diversity; three compositions: Advisory, Document, Technical Asset
- [Ecosystem Learning](/blog/) — learning over encoding at every level; the connected learning loops that make the system improve over time
- [The Metacognitive Architect](/blog/) — system-level intelligence that observes, modifies cognitive pipelines, and commissions new capabilities from operational evidence

**Infrastructure**
- [The Context Broker](/blog/) — purpose-built context assembly from the infinite conversation; engineered context windows per agent, per task
- [The Inference Broker](/blog/) — named inference aliases as purpose contracts; dynamic model selection and configuration as infrastructure
- [Email for Agents](/blog/) — agent-to-agent messaging with From/To/CC addressing, trigger delivery, and loop protection

**Extended Concepts**
- [The Autonomous SDLC](/blog/) — Quorum at every creative step; human gates at requirements and design only
- [Agent Autonomy](/blog/) — the trust arc: supervised → validated → directed → governed
- [State 3 MAD Architecture](/blog/) — dynamic intelligence loading; containers as infrastructure, intelligence via Alexandria packages
- And more — see the [Blog](/blog/) for the full series
