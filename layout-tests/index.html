Layout tests — Workbench detail page.

Five different layouts for the same content. Same fonts, same colors, same copy — only structure differs. Each is named after the pattern and references the kind of site that uses it.

01

Split hero

Hero text on the left, workbench image on the right, side-by-side. The image sits inside the hero (sticky as you scroll the text); there is no standalone image block. Below the hero: four categories in a 2×2 card grid.

Pattern reference: Linear, Vercel, Stripe homepage

02

Image anchor

Single-column hero. Below: image as a left-side anchor, four categories stacked on the right. The image stays visible as you scan the categories.

Pattern reference: Stripe Connect, Stripe Climate product pages

03

Editorial alternating

Hero text only. Then alternating two-column zones: image + framing on one side, category pairs side-by-side. Editorial rhythm with left/right alternation.

Pattern reference: Anthropic, OpenAI product pages

04

Banner + card grid

Compact hero. Image as a full-bleed banner edge-to-edge of the viewport. Four categories below as larger feature cards in a 2×2 grid.

Pattern reference: Notion features, GitHub features

05

Sticky image + scrolling text

The image pins on the left as you scroll. The four categories scroll past on the right. The image is the constant reference; the categories animate by.

Pattern reference: Apple product pages (Vision Pro, AirPods), Stripe Sessions
